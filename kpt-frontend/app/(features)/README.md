## 各フィーチャーの詳細

### 🤖 AI Insights (`/ai-insights`) - プロプラン限定

**機能**: AI分析レポート

- KPTデータを基にした深い分析レポート
- 強み・改善点の自動識別
- 感情トレンド分析とパターン認識
- 実行可能なインサイトと改善提案
- 詳細なチャートとビジュアライゼーション

**特徴**:
- タブ形式での分析結果表示（概要、強み、改善点、感情、提案）
- 期間選択による柔軟な分析
- レポートのダウンロード機能
- AIによる自動分析生成

### 🗺️ Learning Roadmap (`/learning-roadmap`) - プロプラン限定

**機能**: パーソナライズ学習ロードマップ

- KPTデータに基づく個別学習計画の自動生成
- スキルレーダーチャートによる現状可視化
- 段階的学習経路の提示
- 進捗トラッキングとマイルストーン管理
- 学習リソースの推奨

**特徴**:
- 学習目標と強度レベルの選択
- スキル分析とギャップ特定
- 週次推奨アクションの提供
- 学習リソースライブラリ統合

### 🗓️ Calendar (`/calendar`)

**機能**: KPT スケジュール管理

- KPT の振り返りスケジュールをカレンダー形式で表示
- Keep、Problem、Try の予定を色分けして管理
- 統計情報の表示（各タイプの予定数）

### 📞 Contact (`/contact`)

**機能**: お問い合わせ

- ユーザーからの問い合わせフォーム
- サポートチームへの連絡手段

### 📊 Dashboard (`/dashboard`)

**機能**: メインダッシュボード

- KPT 管理の中心となるページ
- 各機能へのナビゲーション
- 統計情報とクイックアクセス

### 🩺 Diagnosis (`/diagnosis`)

## プロプラン機能

KPT Connect では、プロプランユーザー向けの高度なAI機能を提供しています：

### AI分析レポート
- **深い洞察**: KPTデータからAIが抽出する詳細な分析
- **パターン認識**: 感情や行動パターンの自動識別
- **改善提案**: 具体的で実行可能な改善アクション
- **トレンド分析**: 時系列での成長追跡

### 学習ロードマップ
- **パーソナライズ**: 個人の強み・弱みに基づく学習計画
- **スキルマッピング**: 現在のスキルレベルと目標のギャップ分析
- **段階的成長**: 無理のない学習ステップの提示
- **リソース統合**: 推奨書籍、コース、ツールの提案

## 開発ガイドライン

### フィーチャー追加時の規則

1. **ディレクトリ命名**: kebab-case（例: `user-settings`）
2. **必須ファイル**: 各フィーチャーには必ず`page.tsx`を含める
3. **コンポーネント分離**: 複雑な機能は別コンポーネントに分離
4. **型定義**: TypeScript の型定義を適切に行う
5. **プロプラン機能**: プロプラン限定機能は適切なアクセス制御を実装

### ファイル構成例

```
new-feature/
├── page.tsx                    # メインページ（必須）
├── components/                 # フィーチャー固有のコンポーネント
│   ├── FeatureComponent.tsx
│   └── SubComponent.tsx
├── hooks/                      # フィーチャー固有のカスタムフック
│   └── useFeature.ts
├── types/                      # 型定義
│   └── index.ts
└── utils/                      # ユーティリティ関数
    └── helpers.ts
```

### プロプラン機能の実装ガイドライン

1. **アクセス制御**: 必ずプロプランのアクセス確認を実装
2. **エラーハンドリング**: 適切なエラーメッセージとフォールバック
3. **パフォーマンス**: 重い処理は適切にローディング状態を表示
4. **UI/UX**: プロプラン機能であることを明確に表示

### コメント規則

- ファイル先頭に JSDoc コメントを記載
- 機能の概要、使用例を含める
- 日本語でのコメントを推奨
- プロプラン機能には `@access プロプラン限定機能` を明記

### ルーティング

- Next.js App Router の規則に従う
- `(features)`グループ内のページは`/feature-name`でアクセス可能
- 動的ルートは`[param]`形式で定義

## 注意事項

- 新しいフィーチャーを追加する際は、この README も更新してください
- 共通で使用するコンポーネントは`/components/ui`に配置してください
- フィーチャー固有のコンポーネントは各フィーチャーディレクトリ内に配置してください
- API との連携が必要な場合は、適切なエラーハンドリングを実装してください
- プロプラン機能では、適切な課金状態の確認とアクセス制御を実装してください
